# Zero-Date Preservation

Zero-date preservation means the project records its earliest intent before implementation details overwrite the original reason for being.

## Preserved understanding

- The project values **time** as the one resource that cannot be reclaimed.
- The workspace should favor useful capture over slow perfection.
- The MUD must remain readable without a proprietary application.
- Publishing should be routine, not a separate ceremony.

## Practice

When new context appears:

1. Add it to the closest note.
2. Link it to related notes with `[[wiki links]]`.
3. Record unresolved tension in a decision or inbox note.
4. Let the build publish it through the same pipeline.
